Список фич движка

О документе

Для внутреннего пользования (только для программистов).

Список требований и тестов.

Основные

Object-Oriented Design.

All code written in C++ using Visual Studio 2005.

 

Движок делится на несколько логических частей:

Core

Отвечает за связь, инициализацию модулей, [multithreading]. Передаёт управление движку для обработки мира игры на короткое время.

1.            [зациклившаяся часть движка не должна повесить весь движок.]

a.       [Процессоры должны нагружаться равномерно]

b.      [Процессор не должен простаивать]

2.            [консоль позволяет узнавать и изменять большинство параметров всех объектов и движков на лету]

1.            лог-файл. ключевые слова для фильтрации сообщений.

2.            [встроенные скриптовой язык для описания поведения объектов (python, ruby).]

3.            [права скрипта (права доступа).]

4.            в лог выводится информация о системе.

5.            конфиги. Пресеты. Иерархия пресетов. Относительные и абсолютные изменения параметров во внутренних пресетах.

 

Objects

1.            Объекты могут быть сдвинуты, повёрнуты, отмасштабированы

2.            Иерархия объектов. Все трансформации родителя передаются и детям.

3.            [Движок должен оперировать только с той информацией, которая ему нужна. Если каркас объекта не попадает в камеру, то не стоит узнавать точную форму объекта.]

 

Graph Engine

Отвечает за отрисовку переданных ему объектов. Создаёт окно, инициализирует графику. Возможны несколько реализаций движка (OpenGL, D3D, Software).

Поддержка OpenGL 2.0 (Минимальная функциональность на OpenGL 1.1, средняя на OpenGL 1.3 и полная на OpenGL 2.0)

 

 

1.            [Рендеринг одного мира в двух окнах]

2.            [Отсечение невидимых объектов.]

3.            У каждого треугольника свой материал.

4.            В материале указывается:

1.            diffuse текстура (color map)

2.            микротекстура (diffuse2_map)

3.            normal or height map for bumb

4.            emmision

5.            specular_map + степень блика

6.            env_map (карта отражений)

7.            blending – полупрозрачность

8.            трансформация текстурных координат

5.            Динамическое попиксельное освещение (4-8 источников освещения на каждый объект)

6.            Тени (shadow volumes)

7.            [Туман]

8.            Объект может состоять из набора спрайтов. Спрайт может быть выровнен по

a.       Перпендикулярен экрану

b.      По Z-оси (дял кустов)

c.       Перпендикулярен камере   

d.      Фиксированное положение

9.            текстуры

1.            [программно генерируемые]

2.            [анимированные (видеотекстуры)]

3.            [преломление изображения (стекло, огонь)]

4.            [мониторы с изображением с камер (render to texture)]

10.        изменение разрешения экрана на лету

1.            [fullscreen to window mode]

2.            [выставление refresh rate]

11.        [Всевозможные настройки графики (сглаживание, antialiasing, etc)]

12.        [В правильном порядке выводить полупрозрачные полигоны]

13.        [восстанавливать контекст рендеринга при его потере]

14.        [при минимизировании окна (потере фокуса) отпускать мышь и ставить на паузу]

Physic Engine

1.      Кинематика

1.            двигаться линейно

2.            по траектории

3.            вращяться

4.            изменять размер

5.            можно применять эти трансформации по очереди с произвольной задержкой

2.      Система частиц

1.            Способы взаимодействия

1.      LJ

2.      gravity

3.      spring

4.      sprite vs polygon

2.            Отрисовка

1.      спрайты

2.      произвольные объекты, в том числе и другие системы частиц

3.      [meatballs]

3.            Физика твёрдых тел

a.       [Ageia]

4.            Бонусы

a.       моделирования физики

b.      [разделение пространства на домены]

c.       [слипание частиц]

d.      [Различная дельта времени для просчитываемых частиц]

e.       [Может изменять форму объектов (например, от попадания пули). Разделение и разрушение объектов]

Audio Engine

1.      Звуковой двигатель на основе OpenAL 1.0

2.      3D звук

3.      Один слушатель

4.      Один или много источников

5.      Эффект Доплера

[6.      Отражения звука первого порядка на основе виртуальных источников]

7.      Поддержка EAX 2.0

8.      Реверберация на основе EAX 2.0
