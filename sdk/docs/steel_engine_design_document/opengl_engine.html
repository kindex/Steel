OpenGL Engine (Графический движок)

OpenGL имеет множество расширений, которые могут как поддерживаться видеокартой, так и не поддерживаться. OpenGL Engine предоставляет свой интерфейс доступа к возможностям видео-ускорителя без вникания в технические детали реализации, версии и производителя драйвера.

OpenGL versions

Основные версии OpenGL:

1.0: Минимальная функциональность. Сейчас нет таких машин, которые поддерживают только 1.0. Но для общего стиля эта версия оставлена.

1.1: glVertexPointer, Image object

1.3: Mulitexture, простой bump (dot3), cube maps

1.5: Vertex Buffer Object

2.0: GLSL. Именно на эту версию сделан упор. Остальные версии нужны для того, чтобы на старых машинах движок хоть как-то работал бы.

В связи с неоднозначности версии (версия может быть 1.5, но с GLSL) движок содержит указатели на функции, реализующие различные из сервисов. Например, указатель на DrawTriangles может содержать реализацию для 1.0, 1.1 и 1.5 версий.

Interface

Клиент графического движка не может напрямую использовать функции для отрисовки. Он может использовать только инрефейсы Engine и GraphInterface. Они в основном сообщают движку о том, как выглядит объект, а движок уже сам наилучшим образом рисует его (если его надо рисовать).

В этот интерфейс входит:

Форма объекта: vertexes, faces (triangles, quads), texture coordinates, normals, lines (для отладки), AABB

Материал для набора граней

Для чтения доступны координаты камеры и информация о времени и длительности кадра (ProcessInfo).

Эту информацию можно сообщать движку в любое время, начиная сразу же после момента добавления в движок (inject – beforeInject).

Process

Отрисовка одного кадра происходит внутри process().

Вначале обновляется информация. В это время каждому обрабатываемому объекту посылается сообщение UpdateInformation, и при желании он может обновить информацию. Обычно меняется положение объекта, хотя может изменится как форма, так и материал. Так как материал содержит текстуры, то таким образом можно добиться видеотекстур.

После этого кешируются переменные из конфига. Между кадрами можно менять значения этих переменных, например, включая отладочный вывод (drawWire).

Порядок отрисовки сцены (с тенями).

1.      заполняем z-buffer объектами, которые принимают тени (material.catchShadows) + статическое освещение. [replace]

2.      For each light with castShadows = true

a.       применяем Volume Shadows для заполнения stencil-buffer (material.dropShadows = true).

b.      закрашиваем ну часть экрана, что не попадает в тень от этого фонаря. [add]

3.      рисуем оставшиеся объекты (material.dropShadows = true) освещенные фонарями без castShadows + статическое освещение. [add]

 

Результат может быть помещён либо на экран, либо в текстуру (зеркало, монитор), которую можно наклеить на стену или применить постэффект.
