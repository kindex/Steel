Config (Конфиги)

Этот формат файла подходит как для описания сцены, так и для любых конфигов, материалов, анимации, моделей и т.д.

Стандартное расширение файла: *.conf

Требования

Текстовый формат.

Вложенность настроек.

Шаблонность. Массив объектов задаётся первым объектом, а в остальных только изменения относительно первого, или вообще любого другого. Таким способом удобно изменять материал, меняя в нём только текстуру.

Без типизации. Объект сам может распознать нужные настройки из строки.

Includes.

Однозначность.

Расширение конфигов своими типами и парсерами.

Синтаксис

В основе конфига лежит понятие config (значение переменной). Всё содержимое файла является value (исключая комментарии).

<config> ::= [<tag>] <value> | <value>

<value> ::= null

<value> ::= default

<value> ::= <nuber>

<value> ::= ‘<string>’ | “<string>”

<value> ::= {<struct>} | (<array>)

<array> ::= <config> <array> | <config>,<array> |<empty>

<struct> ::= <struct_element> <struct > | <struct_element>;<struct> | <empty>

<struct_element> ::= <alpha> = <config>

 

<tag> ::= <templates>

<templates> ::= <empty> | <external_id>,<inherits>

<external_id> ::= <alpha> | <path to config>#<alpha> | <path to config>#

 

Комментарии в стиле //, /* и #

String – строка, стоящая из любых символов. Символы начала строки ‘ и “ и сам экран \ экранируются при помощи \.

Number – целое или дробное число в стандартной форме или експоненциональной.

alpha - английские буквы и символ подчёркивания, или цифры, начиная со второго символа. Case sensitive.
