О документе

В этом документе описывается порядок и правила работы команды: SVN, встречи, стиль, правила тестирования?

SVN

Исходный код проекта и минимальный набор ресурсов хранится в двух SVN-репозиториях (Subversion). Адрес репозиториев:

svn+ssh://<user>@kindex.lv/var/svn/steel/trunk

svn+ssh://<user>@kindex.lv/var/svn/steel.res/trunk

Доступ через 2222 порт SSH.

SVN_SSH="ssh -p 2222" svn checkout svn+ssh:// <user>@kindex.lv/var/svn/steel/steel/trunk steel

В steel храним текстовые файлы (исходники, доки, конфиги) и бинарники < 1 Kb. В steel.res всё остальное, в основном большие бинарные файлы. Структуры обоих репозиториев повторяют друг друга.

Структура рабочей копии:

·        steel_project – корень проектов

·        > steel

·        > steel.res

Внутренняя структура:

·        Docs – основная документация по всем проектам

·        Sdk – libs, dlls, документация для создания проектов на основе Steel Engine.

·        Steel – исходники движка + unit-tests, внутренная документация

·        Tasks – список заданий (свободные задания; кто, что сейчас делает)

·        Temp – дира для временных файлов. Не должны быть в репозитории.

·        Demo – accept-тесты, примеры использования движка.

·        Wurps – W.U.R.P.S project

·        Pool – Noncence Pool project

 

 

При добавлении нового файла не забывайте вставлять шапку, которая описана в соглашении о стилях. Создавайте текстовый файл в кодировке UTF-8.

В начале каждого h или cpp файла должен быть #include "steel.h"

Добавляя файл в проект, добавляйте его во все проекты:

·        steel.vcproj

·        steel.dev

·        SConstruct

·        steel_lib.vcproj

·         steel_lib.dev

·         test_steel.vsproj

·         test_steel.dev

Перед каждым коммитом проверяёте целостность коммита (все ли файлы добавлены, нет ли случайных изменений, добавлены ли новые файлы в проектные файлы). Полностью перекомпилируйте проекты test_steel и запустите его. Ошибок не должно быть. После коммита сделайте update в чистую директорию и повторите тесты. После этого добавьте сообщение о сделанных изменениях в чат. После этого надо отпраздновать конец процесса (самый важный пункт).
