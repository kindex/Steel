Code Conventions

by Kuzmi4

20060726

 

Главной целью соглашений о программном коде является улучшение читаемости и поэтому понимания и удобства обслуживания программного кода.

 

 

Соглашения о присваивании имен

 

  • Имена, представляющие типы должны начинаться с большой буквы. Первая буква каждого внутреннего слова должна быть большой.

 

Line, SoundInfo, ConvexFigure...

 

  • Переменные должны начинаться с маленькой буквы. Первая буква каждого внутреннего слова должна быть большой.

 

line, myAccount, centerOfGravity, theRectangle...

 

  • Константы (включая перечислительные значения) должны состоять только из больших букв, разделяя внутренние слова знаком подчеркивания.

 

MAX_ITERATIONS, COLOR_RED, PI...

 

  • Имена, соответствующие методам или функциям должны быть глаголами и пишутся с маленькой буквы. Первая буква каждого внутреннего слова должна быть большой.

 

calculateArea(), getLength(), setWidth()...

 

·        Имена, соответствующие названиям именованной области видимости (namespace), пишутся маленькими буквами.

 

model::analyzer, io::iomanager, common::math::geometry

 

·        Имена, соответствующие названиям шаблонов (template) должны состоять из одной большой буквы.

 

template<class T> ...

template<class C, class D> ...

 

·        Сокращения и акронимы не должны писаться большими буквами, когда используются как имена.

 

exportHtmlSource(); // NOT: exportHTMLSource();

openDvdPlayer();    // NOT: openDVDPlayer();

 

·        All names should be written in English.

 

filename;   // NOT: imjaFaila;

 

English is the preferred language for international development.

 

·        Имена объектов являются неявными и не должны использоваться в именах методов.

 

line.getLength();   // NOT: line.getLineLength();

 

·        Имена переменных должны быть короткие и наглядные (имеющие ясную цель). Однобуквенные переменные не должны использоваться. В виде исключения – временные переменные. Общепринятые имена временных переменных являются:

§         i, j, k, m, n – для целых значений

§         c, d, e – для символов

 

 

Специальные соглашения о присваивании имен

 

·        Если методы изменяют свойства класса, то они должны называться

 

setXXX();

 

      Если методы возвращают конкретные свойства класса, то они должны называться:

 

getXXX(); - для нелогических значений

 

isXXX(); - для логически значений

 

·        Терм compute может быть использован, когда что-нибудь вычисляется.

 

valueSet->computeAverage();

 

matrix->computeInverse();

 

·        Терм find может быть использован, когда что-нибудь ищется

 

vertex.findNearestVertex();

 

matrix.findMinElement();

 

·        Терм init может быть использован, когда что-то устанавливается

 

printer.initializeFontSet();

 

Сокращение init должно избегаться.

 

·        Переменные, соответствующие компонентам GUI должны быть с приписанным в конце именем компоненты.

 

mainWindow, propertiesDialog, widthScale, loginText,

leftScrollbar, mainForm, fileMenu, minLabel, exitButton, yesToggle etc.

 

·        Множественное число должно использоваться, когда имя соответствует множеству объектов.

 

vector<Point>  points;

int            values[];

 

·        Приставка n должна использоваться для переменных, означающих количество объектов.

 

nPoints, nLines

 

·        Приставка is должна использоваться для логических переменных и методов.

 

isSet, isVisible, isFinished, isFound, isOpen

 

Есть несколько альтернатив, которые в некоторых ситуациях подходят лучше. Это приставки has, can и should.

 

bool hasLicense();

bool canEvaluate();

bool shouldSort();

 

·        Следующие имена должны использоваться для соответствующих им операций:

 

get/set, add/remove, create/destroy, start/stop, insert/delete, increment/decrement, old/new, begin/end, first/last, up/down, min/max, next/previous, old/new, open/close, show/hide, suspend/resume, etc.

 

·        Сокращения в именах должны избегаться.

 

computeAverage();   // NOT: compAvg();

 

Различаются два типа слов. Первый – обычные слова из словаря. Они никогда не должны сокращаться. Никогда не пишите:

 

cmd   вместо command

cp    вместо copy

pt    вместо point

comp  вместо compute

init  вместо initialize

etc.

 

Второй тип – известные и общепринятые сокращения и акронимы. Эти фразы должны сокращаться. Никогда не пишите:

 

HypertextMarkupLanguage вместо html

CentralProcessingUnit вместо cpu

etc.

 

·        Избегайте специальных названий указателей.

 

Line* line; // NOT: Line* pLine;

              // NOT: LIne* linePtr;

 

·        Избегайте отрицательных логических выражений.

 

bool isError; // NOT: isNoError

bool isFound; // NOT: isNotFound

 

·        Перечислительные константы могут быть с приставкой, которая отображает общий тип.

 

enum Color { COLOR_RED, COLOR_GREEN, COLOR_BLUE };

 

·        Функции (методы, возвращающие что-либо) должны называться согласно тому, то они возвращают; процедуры (void-методы) должны называться согласно тому, что они делают.

 

 

 

Соглашения о файлах

 

·        C++ header-файлы должны иметь расширение .h. Исходные файлы должны иметь расширение .cpp.

 

MyClass.cpp, MyClass.h

 

·        Класс должен декларироваться (declaration) в заголовочном файле и определяться (definition) в исходном файле, где название файлов должно соответствовать названию класса.

·        Все определения должны оставаться в исходном файле.

 

class MyClass

{

public:

       int getValue () {return value;}  // NO!

...

 

private:

       int value;

}

 

·        Содержание файла должно храниться в 80 колонках.

·        Неполнота разделенных строк должна быть очевидной.

 

totalSum = a + b + c +

           d + e;

 

function (param1, param2,

          param3);

 

setText ("Long line split"

         "into two parts.");

 

for (int tableNo = 0; tableNo < nTables;

     tableNo += tableStep) {

  ...

}

 

Строки переносятся, когда длина строки достигает 80 символов. Перенос осуществляется:

§         после запятой

§         после оператора

§         новая строка располагается под началом выражения предыдущей строки.

 

·        Заголовочные файлы должны содержать конструкцию

#ifndef COM_COMPANY_MODULE_CLASSNAME_H

#define COM_COMPANY_MODULE_CLASSNAME_H

  :

#endif

 

·        Include-выражения должны быть отсортированы и сгруппированы. Сортировка происходит по их иерархическому положению в системе, при этом низкоуровневые файлы прикрепляются первыми.

 

#include <fstream>

#include <iomanip>

 

#include <qt/qbutton.h>

#include <qt/qtextfield.h>

 

#include "com/company/ui/PropertiesDialog.h"

#include "com/company/ui/MainWindow.h"

 

·        Include-выражения должны находиться только вначале файла

 

 

 

Соглашения о выражениях

 

      Типы

     

·        Части класса должны быть отсортированы в следующем порядке: public, protected и private.

·        Преобразования типов должно осуществляться явно.

 

floatValue = static_cast<float>(intValue);

// NOT: floatValue = intValue;

 

  Переменные

 

·        Переменные должны быть проинициализированы там, где они объявляются.

·        Использование глобальных переменных должно быть минимизировано.

·        Переменные в классе никогда не должны декларироваться как public.

·        Ссылки и указатели должны иметь свой ссылочный знак рядом с типом.

 

float* x; // NOT: float *x;

int& y;   // NOT: int &y;

 

·        Неявный тест на 0 не должен нигде использоваться кроме как в логических выражениях.

 

if (nLines != 0)  // NOT: if (nLines)

if (value != 0.0) // NOT: if (value)

 

Циклы

 

·        Только управляющее выражение может быть включено в конструкцию for( )

 

sum = 0;  // NOT: for (i = 0, sum = 0; i < 100; i++)

for (i = 0; i < 100; i++)  // sum += value[i];

sum += value[i];

 

·        Переменные должны быть инициализированы непосредственно перед циклом.

 

isDone = false;       // NOT: bool isDone = false;

while (!isDone)       //      :

{

  :                   //      while (!isDone)

}                            {

                      //        :

                      //      }

 

·        do-while циклы рекомендуется избегать.

·        Форма while(true) должна использоваться для бесконечных циклов.

 

while (true) {

  :

}

 

for (;;) {  // NO!

  :

}

 

while (1) { // NO!

  :

}

 

      Условные выражения

 

·        Сложные условные выражения должны избегаться. Для этого необходимо ввести временные переменные.

 

bool isFinished = (elementNo < 0) || (elementNo > maxElement);

bool isRepeatedEntry = elementNo == lastElement;

if (isFinished || isRepeatedEntry)

{

  :

}

 

// NOT:

if ((elementNo < 0) || (elementNo > maxElement)||

     elementNo == lastElement)

{

  :

}

 

·        Условное выражение должно быть помещено на отдельной строке.

 

if (isDone)       // NOT: if (isDone) doCleanup();

  doCleanup();

 

·        Исполняемые выражения нельзя использовать в условных выражениях.

 

File* fileHandle = open(fileName, "w");

if (!fileHandle)

{

  :

}

 

// NOT:

if (!(fileHandle = open(fileName, "w")))

{

  :

}

 

      Разное

 

·        Числа, отличные от 0 и 1 должны определяться как константы.

·        Константы с плавающей точкой должны всегда писаться с точкой и по крайней мере одним знаком после нее.

 

double total = 0.0;    // NOT:  double total = 0;

double speed = 3.0e8;  // NOT:  double speed = 3e8;

 

double sum;

:

sum = (a + b) * 10.0;

 

·        Константы с плавающей точкой должны всегда писаться с цифрой перед точкой.

 

double total = 0.5;  // NOT:  double total = .5;

 

·        Goto использовать нельзя.

·        “0” должен использоваться вместо “NULL”

 

 

 

Соглашения о расположении и комментариях

 

      Расположение

 

·        Основной отступ должен быть равен двум пробелам.

 

for (i = 0; i < nElements; i++)

  a[i] = 0;

 

·        Расположение блока

 

while (!done)

{

  doSomething();

  done = moreToDo();

}

 

·        Декларация класса должна иметь следующую форму:

 

class SomeClass : public BaseClass

{

  public:

    ...

 

  protected:

    ...

 

  private:

    ...

}

 

·        Определения методов должны иметь следующую форму:

 

void someMethod()

{

  ...

}

 

·        Выражения if-else должны иметь следующую форму:

 

if (condition)

{

  statements;

}

 

if (condition)

{

  statements;

}

else

{

   statements;

}

 

if (condition)

{

  statements;

}

else if (condition)

{

  statements;

}

else

{

  statements;

}

 

·        Выражение for должно иметь следующую форму:

 

for (initialization; condition; update)

{

  statements;

}

 

·        Пустое выражение for должно иметь следующую форму:

 

for (initialization; condition; update)

  ;

 

·        Выражение while должно иметь следующую форму:

 

while (condition)

{

  statements;

}

 

·        Выражение do-while должно иметь следующую форму:

 

do

{

  statements;

}

while (condition);

 

·        Выражение switch должно иметь следующую форму:

 

switch (condition)

{

  case ABC :

    statements;

    // Fallthrough

 

  case DEF :

    statements;

    break;

 

  case XYZ :

    statements;

    break;

 

  default :

    statements;

    break;

}

 

·        Выражение try-catch должно иметь следующую форму:

 

try

{

  statements;

}

catch (Exception& exception)

{

  statements;

}

 

·        Простые if-else, for или while выражения могут быть написаны без скобок.

 

if (condition)

  statement;

 

while (condition)

  statement;

 

for (initialization; condition; update)

  statement;

 

Рекомендуется все же использовать скобки даже в таких случаях.

 

 

      Пробелы

 

·        Операторы должны быть окружены пробелами

·        За резервированными словами должен следовать пробел

·        За запятыми должен следовать пробел

·        Двоеточие должно быть окружено пробелами

·        После точек с запятой в for выражениях должны быть пробелы

 

a = (b + c) * d; // NOT: a=(b+c)*d

 

while (true)   // NOT: while(true)

{

  ...

 

doSomething(a, b, c, d);  // NOT: doSomething(a,b,c,d);

 

case 100 :  // NOT: case 100:

 

for (i = 0; i < 10; i++) {  // NOT: for(i=0;i<10;i++){

  ...

 

·        Логические части блока должны разделяться пустой строкой.

 

Matrix4x4 matrix = new Matrix4x4();

 

double cosAngle = Math.cos(angle);

double sinAngle = Math.sin(angle);

 

matrix.setElement(1, 1,  cosAngle);

matrix.setElement(1, 2,  sinAngle);

matrix.setElement(2, 1, -sinAngle);

matrix.setElement(2, 2,  cosAngle);

 

multiply(matrix);

 

·        Методы должны отделяться друг от друга тремя пустыми строками.

 

 

Комментарии

           

·        Использование комментариев должно быть минимизировано. Код должен быть ясный с правильными именами и логической структурой.

·        All comments should be written in English. In an international environment English is the preferred language.

·        Использовать // для всех комментариев, включая многострочные комментарии.

·        Комментарии должны находиться относительно их позиции в коде

 

while (true)        // NOT:  while (true)

{                            {

  // Do something             // Do something

  something();                  something();

}                           }
