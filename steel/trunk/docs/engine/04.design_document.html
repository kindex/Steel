Physic Engine

Physic Interface [kindex]

Любой объект, который должен обрабатываться стандартными физическими движками, должен поддерживать этот интерфейс (быть унаследованным от PhysicInterface). Объект сам отвечает за хранение всех параметров (конфигурация). Объект может брать парамерты из других мест, например коллекции ресурсов. Объект в определённых случаях может менять свои параметры, например скорость или форму.

До добавления в [физический] движок объект может как угодно менять свои параметры. Непосредственно перед добавлением в движок вызывается функция bool beforeInject(). Объект не добавляется, если возвращается false. После удаления из движка вызывается процедура void afterRemove(). После удаления из движка обхект может как угодно менять свою конфигурацию.

Объект не может быть добавлен сразу в два движка [Параметры могут кешироваться и второй движок не узнает изменения – произойдёт рассинхронизация]
