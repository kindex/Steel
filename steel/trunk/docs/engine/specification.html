Спецификация движка

Для внутреннего пользования (только для программистов).

Список требований и тестов.

 

Движок делится на несколько логических частей:

Core

Отвечает за связь, инициализацию модулей, multithreading. Передаёт управление движку для обработки мира игры на короткое время.

1.      зациклившаяся часть движка не должна повесить весь движок.

2.      Процессоры должны нагружаться равномерно

3.      Процессор не должен простаивать

4.      кросплатформенность

5.      консоль позволяет узнавать и изменять большинство параметров всех объектов и движков на лету

1.      лог-файл. ключевые слова для фильтрации сообщений.

2.      встроенные скриптовой язык для описания поведения объектов

3.      права скрипта (права доступа)

4.      в лог выводится информация о системе

6.      конфиги

 

Graph Engine

Отвечает за отрисовку переданных ему объектов. Создаёт окно, инициализирует графику. Несколько реализаций движка (OpenGL, D3D)

 

1.      Рендеринг одного мира в двух окнах

2.      Объекты могут быть сдвинуты, повёрнуты, отмасштабированы

3.      Иерархия объектов. Все трансформации родителя передаются и детям.

4.      Отсечение невидимых объектов.

5.      Должен оперировать только с той информацией, которая ему нужна. Если каркас объекта не попадает в камеру, то не стоит узнавать точную форму объекта.

6.      У каждого треугольника свой материал.

7.      В материале указывается:

1.      diffuse текстура (color map)

2.      bump (normal map)

3.      illuminating

4.      карта преломления (reflect)

5.      shader

8.      Динамическое освещение

9.      Тени

10.  Туман

11.  Объект может состоять из набора спрайтов

12.  Минимальная функциональность на OpenGL 1.1, средняя на OpenGL 1.3 и полная на OpenGL 2.0

13.  текстуры

1.      программно генерируемые

2.      анимированные (видеотекстуры)

3.      преломление изображения (стекло, огонь)

4.      мониторы с изображением с камер (render to texture)

14.  изменение разрешения экрана на лету

1.      fullscreen to window mode

2.      выставление refresh rate

15.  Всевозможные настройки графики (сглаживание, antialiasing, etc)

16.  В правильном порядке выводить полупрозрачные полигоны

Physic Engine

1.      Кинематика

1.      Движение по прямой линии

2.      Свободное падение

1.      Движение по заранее просчитанной траектории (по рельсам)

2.      Придание ускорения телам

3.      Неподвижные объекты (пол, стены)

2.      Определение коллизий

1.      для частиц (LJ, gravity)

2.      для сфер

3.      для многогранников

4.      для каждого типа с каждым

3.      Динамика

1.      Изменение скорости тел по законам Ньютона при ударе

2.      Изменение скорости и вращения тел при ударе

4.      Иерархия

1.      кластеры (комнаты). Переход в соседние комнаты.

2.      домены коллизий

3.      прилипание объектов к родителю. Сила связи

5.      Может изменять форму объектов (например, от попадания пули). Разделение и разрушение объектов

Audio Engine

1.      Многоканальный звук

2.      3D звук

3.      эффект движения

4.      размер помещения и тип стен влияют на звучание

 

Resources

1.      Загрузка следующих типов ресурсов

1.      image

1.      bmp

2.      tga

3.      jpeg

4.      png

5.      from quake wad-file (pcx)

2.      model

1.      3ds

2.      lightwave

3.      ase (text format)

4.      lightmap/3Dmax rendering

5.      quake map (text format)

6.      quake bsp

3.      audio

1.      wav

2.      ogg

3.      загрузка только нужной части звука

4.      material

5.      level – набор моделей, триггеров и правил поведения объектов. Пресеты. Иерархия пресетов. Относительные и абсолютные изменения параметров во внутренних пресетах.

6.      конфиги. Приоритет настроек.

2.      загрузка из файловой системы, архива (zip, tar)

3.      выгрузка неиспользуемых ресурсов

4.      все ресурсы идентифицируются строчкой, которая в большинстве случаев является путём к файлу без расширения.

5.      коллекция ресурсов пытается загрузить ресурс всеми доступными загрузчиками для этого типа ресурса

Objects

1.      полигональная модель, универсальная физика

a.       изменяемое количество детей

b.      изменяемая форма и количество полигонов

2.      системы частиц

a.       различные типы эмиттера

b.      различные типы физики

                                                                          i.      универсальная физика

1.      LJ

2.      gravity

3.      пружинная связь

                                                                         ii.      простая быстрая физика для графических систем частиц

c.       различный рендеринг

                                                                          i.      sprite system

                                                                         ii.      каждая частица – объект

                                                                       iii.      meatballs

3.      скелетная анимация
