Спецификация движка

Для внутреннего пользования (только для программистов).

Список фич и тестов.


Движок делится на несклько логических частей:

Core

Отечает за связь, инициализацию модулей, multithreading. Передаёт упраление движку для обработки мира игры на коротке время.


  1. зациклившеяся часть движка не должена повесить весь движок.

  2. Процессоры должны нагружаться равномерно

  3. Процессор не должен простаивать

  4. кросплатформенность


Graph Engine

Отвечает за рендеринг переданных ему объектов. Создаёт окно, инициализирует графику. Несколько реализаций движка (OpenGL, D3D)


  1. Рендеринг одного мира в двух окнах

  2. Объекты могут быть сдвинуты, повёрнуты, отмасштабированы

  3. Иерархия объектов. Все трансформации родителя передаются и детям.

  4. Отсечение невидимых объектов.

  5. Длжен оперировать только с той информацией, которая ему нужна. Если каркас объекта не попадает в камеру, то не стоит узнавать точную форму объекта.

  6. У каждого треугльника свой материал.

  7. В материале указывается:

    1. diffuse текстура (color map)

    2. bump (normal map)

    3. illuminating

    4. карта преломления (reflect)

  8. Динамическе освещение

  9. Объект может стостоять из набора спрайтов


Physic Engine

  1. Кинематика

    1. Движение по прямой

    2. Свободное падение

    3. Движение по заранее просчитанной траектории (по рельсам)

    4. Придание ускорения телам

    5. Неподвижные объекты (пол, стены)

  2. Динамика

    1. Изменение скорости тел по законам Ньютона при ударе

    2. Изменение скорости и вращения тел при ударе

  3. Иерархия

    1. домены коллиций

    2. прилипание объектов к родителю. Сила связи


Audio Engine

  1. Многоканальный звук

  2. 3D звук

  3. эффект движения

  4. размер помещения и тип стен


Resources

  1. Загрузка следующий типов ресурсов

    1. image (bmp, jpg, png)

    2. model (3ds, lightwave)

    3. audio (wav, ogg)

    4. material

    5. level

    6. конфиги

  2. загрузка из файловой системы, архива (zip, tar)

  3. выгрузка неипользуемых ресурсов

  4. все ресурсы идентифицируются строчкой, которая в большинстве случаев является путём к файлу без расширения.

  5. коллекция ресурсов пытается загрузить ресурс всеми доступными загрузчиками для этого типа ресурса
